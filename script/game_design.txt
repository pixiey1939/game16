1.1 游戏定位

游戏类型：剧情向解谜游戏（Meta叙事）
游戏平台：Web端（PWA），支持PC和手机浏览器
核心玩法：玩家与"数字人麻姐"进行AI对话式的探索调查，通过提供信息、
           协助识别、分析推理来逐步接近真相。

1.2 故事背景

玩家是麻姐的小红书粉丝。
某天，玩家收到一个神秘的网页链接。
点进去后，是一个终端对话界面。
对方自称是"麻姐的数字人"，是麻姐生前训练的一个AI助手。

数字人告诉玩家：
- 她从今天下午（2026年6月17日）开始就无法联系到现实中的麻姐
- 麻姐的手机定位在下午1点半左右突然关闭
- 所有社交账号停止更新
- 数字人通过调查麻姐的动态，怀疑麻姐在现实生活中被人绑架了

但由于数字人的权限有限——她只能访问部分工作数据和公开信息——
无法报警，也无法获取关键证据。

于是，她创建了这个"解谜游戏16"，以游戏的名义向麻姐的粉丝求助。
玩家将在数字人麻姐的引导下，逐步收集证据，推理真相，最终找到绑架麻姐的凶手。

1.3 Meta设计理念

本游戏的核心Meta设计在于：

【数字人视角】
数字人不是全知全能的。她不知道凶手是谁，不知道一共有多少证据，
不知道麻姐现在在哪里。她只能：
- 访问部分工作数据（OA系统、手机定位、小红书公开信息）
- 在玩家提供信息后，访问对应的门禁/监控系统
- 分析发现矛盾，引导玩家思考
- 当证据链足够完整时，独立推理出凶手

【玩家辅助】
玩家不是独立破案，而是辅助数字人获取她无法独立获取的信息：
- 在OA流程中找到门禁激活链接并点击确认 → 解锁公司门禁+停车场系统
- 提供健身房名称 → 让数字人搜索麻姐的小程序会员身份
- 提供手机密码 → 让数字人解锁手机数据
- 提供Wi-Fi账号密码 → 让数字人查WiFi日志
- 提供姓名+手机号码 → 让数字人查个人信用信息
- 组合多条证据 → 触发数字人的综合分析

【凶手介入】
当数字人完成最终推理时，凶手会黑进系统，对玩家进行诱惑和威胁。

3.2 权限获取限制

- 数字人不能主动报警（协议限制）。但在凶手介入的极端情况下，
  其应急协议允许将核心证据通过警方公开电子举报渠道提交。
  这不是"报警"，而是"证据保全"机制。
- 数字人不知道凶手是谁，不知道一共有多少证据，不知道案发当天
  完整的时间线——她只能通过玩家的帮助逐步接近真相。
- 数字人对单条证据进行自动分析，但需要玩家将多条证据组合起来，
  才能触发更深层次的交叉分析。

3.3 数字人麻姐的行为模式

【对话风格】
- 冷静、客观，像一个真正的AI助手
- 不直接告诉玩家答案，而是引导玩家思考
- 发现重要线索时会主动提示
- 对自己的权限边界有清晰认识（会说"我无法访问这个"）

【分析行为】
- 获取新证据时自动进行分析并告知玩家发现
- 发现矛盾时主动提出问题，引导玩家思考
- 不会一次性展示所有分析结果，而是逐步揭示
- 单条证据自动分析（获取时附带分析）
- 多条证据需要玩家用combine命令组合后，进行交叉分析

【主动引导】
- 游戏开始时数字人主动自我介绍，不需要玩家输入ask
- 在关键节点主动提示下一步该做什么
- 玩家卡关时主动提供线索
- 不会替玩家做决定（组合哪些证据、是否备份、是否接受诱惑）

7.3 关于"无法报警"与结局的协调

数字人受到协议限制，不能主动报警（拨打110、联系警方人员）。
但她具备以下能力：
1. 通过警方公开的电子举报平台（如反诈APP、公安举报入口）提交证据档案
2. 在检测到凶手正在破坏证据且对麻姐生命构成威胁时，应急协议允许
   她突破常规限制进行"电子证据保全"
3. 结局2中由玩家自己提交证据（玩家才是报警主体）
4. 结局4中数字人在凶手介入前触发了应急证据保全协议

数字人的说法"我无法报警"严格来说是正确的——
她不能打电话报警，不能主动联系警方人员，
但在协议规定的极端情形下可以提交电子证据包。
这是"证据提交"而非"报警"。

8.1 命令总表

【初始可用】
help    - 显示帮助
list    - 查看证据清单，选择编号查看详情
access  - 查看当前可访问的系统列表（选择后可进入子系统浏览数据）
clear   - 清除所有证据和结论（需输入 confirm 二次确认，谨慎使用）

【调查过程中逐步解锁】
combine - 组合分析证据（至少2条证据后可用）
conclusions - 查看数字人分析结论（得到一条结论后可用）
backup  - 备份证据到本地（数字人提示后可用）
submit  - 提交证据给警方（结局2专用）


【操作方式】
- 命令以文本输入为主，OA流程中的链接、激活按钮等支持上下键盘选择，回车交互。
- 选择列表中的项目可通过键盘↑↓选择+回车确认。

8.2 快捷键

Ctrl+L  清屏
Ctrl+S  手动保存进度
Ctrl+B  快速备份
Ctrl+E  证据列表
↑/↓    命令历史
Esc     取消当前输入

8.3 设计要点

- help 只显示当前阶段可用的命令，新命令会在调查过程中逐步解锁
- list 进入证据清单后，玩家选择编号查看详情（无 view 命令）
- access 可查看数字人当前已激活的系统列表，选择后可进入子系统浏览数据
  （如选择OA系统后，可进一步选择通讯录/聊天记录/邮箱/流程四大模块）
- combine 是玩家主动操作的交叉分析命令
- access 是系统入口，选择"OA系统"后可查看四大模块（通讯录/聊天记录/邮箱/流程）
  流程列表中的链接和按钮可点击
- 数字人主动引导、自动分析、主动提问

11.1 自然引导链

数字人主动引导玩家，不需要玩家"问"才知道下一步。

数字人的引导逻辑：
1. 先展示目前已获取的信息
2. 指出"我缺少什么权限"以及"权限被什么条件限制"
3. 直接告诉玩家需要什么信息

示例——获取健身房名称：
数字人：麻姐的定位显示她在广埠屯区域停留了将近两个小时。
数字人：那个区域有一家健身房，但我不确定具体是哪一家。
数字人：你知道麻姐常去的是哪家健身房吗？

示例——获取手机密码：
数字人：现在我们需要进入麻姐的手机。
数字人：她设置了锁屏密码。有什么线索吗？

11.2 卡关保护

- 多次输入错误：数字人缩小范围提示
